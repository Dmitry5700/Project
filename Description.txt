Админка наставника
Чтобы понять лучше см пример фрагмента учебного материала.
Две основных функции: коммуницировать с учащимися, редактировать учебный контент.
Открыть файл json, содержащий курс
Содержимое парсится
Теги с подсветкой

Общий вид формата учебного материала

[
{
type: 'task',

/* Имя задачи */
title: 'Настраиваем окружение',

/* Ключевики задачи для портфолио */
Tags: 'git, html, terminal'

/* Технические ключевики */
Tech: 'git.clone, git.add, git.commit, git.push, terminal'

/* постановка задачи */
description: [%Content%, %Content%, ...]

/* Туторалы */
tutorials: [%Content%, %Content%, …],

/* Подсказки для практики */
practicHints: [%Content%, %Content%, …],

/* Приватный коммент, только для админов */
comment: %Content%
},

{

},

...
]

/* Занимает свой отдельный блок, каждый новый контент показывается на следующем шаге */
%Content% = {

/* Текст с поддержкой нескольких тегов */
/* <a> <author> <img> <code> <listing>, <video> */
src: 'Вы пришли на работу, с чего вы начнете? Конечно с настройки рабочей среды. Так и у нас
Подготавливаем среду, ставим гит.
В качетстве редактора можно использовавать: sublime text 2 
Тебе нужно создать репозиторий на github.com, запушить туда один файл, например readme. И сохранить ссылку в задаче.'

}

Админка должа позволять работать с вышеперечисленными тегами в поле контента.
<author>, <code>, <listing> - выделяем фрагмент текста и просто жмем кнопочку
<a>, <video>, <img> - ставим курсор в нужное место, кликаем тег, в окне вводим ссылку.
img - надо предварительно аплоадить на compilisto.com

Навигация аналогично порталу. Слева колонка задач. При выборе задачи идем влево-вправо кнопками: сначала описание, потом туториал, потом практика с подсказками.
Поля, общие для всей задачи видим всегда: title, tags, tech, comment
Важно: надо стараться переиспользовать код работы с админкой и основной код портала, должно быть много общих функций

Если фрагмент текста выделен, то при нажатии на author открывается маленькое окошечко в котором я могу выбрать роль: manager, coach, designer, tester (radio button например)
